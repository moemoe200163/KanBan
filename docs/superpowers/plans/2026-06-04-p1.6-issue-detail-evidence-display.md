# P1.6 Issue Detail — Evidence Display (HandoffCard scope) Implementation Plan

> **For agentic workers:** REQUIRED SUB-KILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render a completed handoff's typed payload values inside `HandoffCard.vue` via a component-local collapsible body, so a reviewer can see *what was actually delivered* on a handoff — not just the handoff's status badge and lane.

**Architecture:** Sole production-code change is in `src/components/lane/HandoffCard.vue`. A local `ref(false)` (`expanded`) plus two `computed` (`payloadKeyCount`, `showEvidenceToggle`) gate a toggle button (`data-testid="handoff-evidence-toggle"`) and a body block (`data-testid="handoff-evidence-body"`) inserted between the existing payload-key chips and the action buttons. Body renders `handoff.payload` with type-aware formatting: `<ul>` for arrays, integer with `%` suffix for `coverage_pct`, `max-h-48 overflow-y-auto whitespace-pre-wrap` for long/multiline strings, inline text for short strings, `String(value)` fallback. No Pinia, no store changes, no HandoffSection/IssueDetail/API changes. Two new e2e tests in `e2e/handoff-completion.spec.ts` (both desktop-only) cover toggle visibility + expand/collapse on a completed handoff, and toggle absence on a non-completed handoff.

**Tech Stack:** Nuxt 3 / Vue 3 / TypeScript / Pinia (component-local state only), Playwright e2e, Tailwind utility classes.

**Spec reference:** `docs/superpowers/specs/2026-06-04-issue-detail-evidence-display-design.md` (commits `560a90f` + `e2c6d85`).

**Hard guardrails** (pinned by tests + spec):
1. `showEvidenceToggle` is `handoff.status === 'completed' && payloadKeyCount > 0` — toggle must NEVER render for `accepted` / `in_progress` / `blocked` / `cancelled` / `pending`.
2. Body MUST use `<ul>` for arrays — never `JSON.stringify`.
3. Long strings (>= 280 chars OR contains `\n`) MUST use `max-h-48 overflow-y-auto whitespace-pre-wrap` — never clamp / truncate.
4. `coverage_pct` integer values MUST render with a `%` suffix.
5. No emission, no new Pinia state, no HandoffSection.vue changes.
6. **No P2 Artifacts typed seam added this sprint** (no `ArtifactRef` / `IssueEvidence` types in `src/types/`). Spec §3.2 records the shape for the P2 designer; the consumer is a `HandoffCard`-local collapsible body, not a global panel. Defer.

```ts
import { expect, test, type APIRequestContext, type Page } from '@playwright/test'

// Open an issue via the e2e store hook and switch to the Handoffs tab.
// The user-facing path is a card click, but the kanban columns container
// and the review-queue overlay can sit on top of the card's bounding
// box and intercept the click; the store-hook path is what
// e2e/dependency.spec.ts uses for the same reason. This is the tracked
// UX bug noted in the spec §6, not a test methodology preference.
const openIssueAndSwitchToHandoffsTab = async (
  page: Page,
  issueId: string
) => {
  await page.evaluate((id) => {
    const hook = (window as Window).__DEVFLOW_E2E__
    if (!hook) {
      throw new Error(
        '__DEVFLOW_E2E__ not exposed. Confirm NUXT_PUBLIC_E2E=1 is set ' +
        'on the preview server (see e2e/playwright.config.ts webServer.env).'
      )
    }
    const issue = hook.store.getAllIssues.find(i => i.id === id)
    if (!issue) throw new Error(`Issue ${id} not found in store`)
    hook.store.selectIssue(issue)
  }, issueId)
  await expect(page.locator('.issue-detail__panel')).toBeVisible()
  await page.getByRole('button', { name: 'Handoffs' }).click()
}
```

```ts
const relativeTime = computed(() => {
  const d = new Date(props.handoff.createdAt)
  const now = Date.now()
  const diff = now - d.getTime()
  if (diff < 60_000) return 'just now'
  if (diff < 3_600_000) return `${Math.floor(diff / 60_000)}m ago`
  if (diff < 86_400_000) return `${Math.floor(diff / 3_600_000)}h ago`
  return d.toLocaleDateString()
})

// Evidence display — collapsed by default, completed handoffs only.
// Component-local state; never read by parent. Refreshes on prop change
// so a re-fetched Handoff with a populated payload updates the count.
const expanded = ref(false)

const payloadKeyCount = computed(
  () => Object.keys(props.handoff.payload ?? {}).length
)

// accepted / in_progress / blocked / cancelled / pending are
// intentionally NOT eligible — they have no "evidence" to show.
const showEvidenceToggle = computed(
  () => props.handoff.status === 'completed' && payloadKeyCount.value > 0
)
```

```vue
    <!-- Evidence body (only when expanded) -->
    <div
      v-if="showEvidenceToggle && expanded"
      data-testid="handoff-evidence-body"
      class="mb-2 rounded border border-zinc-800 bg-zinc-900/40 p-2 space-y-1.5"
    >
      <div
        v-for="(value, key) in handoff.payload"
        :key="key"
        class="text-[11px]"
      >
        <div class="text-zinc-500 mb-0.5">{{ key }}</div>

        <!-- list[str] (e.g. screenshots, interfaces, acceptance_criteria) -->
        <ul
          v-if="Array.isArray(value)"
          class="list-disc list-inside text-zinc-300 space-y-0.5"
        >
          <li v-for="(item, i) in value" :key="i">{{ item }}</li>
        </ul>

        <!-- number (e.g. coverage_pct) -->
        <div v-else-if="typeof value === 'number'" class="text-zinc-300">
          {{ value }}{{ key === 'coverage_pct' ? '%' : '' }}
        </div>

        <!-- long string (>= 280 chars OR contains newline) -->
        <div
          v-else-if="typeof value === 'string' && (value.length >= 280 || value.includes('\n'))"
          class="text-zinc-300 max-h-48 overflow-y-auto whitespace-pre-wrap
                 bg-zinc-950/40 rounded p-1.5 border border-zinc-800/60"
        >{{ value }}</div>

        <!-- short string -->
        <div v-else-if="typeof value === 'string'" class="text-zinc-300">
          {{ value }}
        </div>

        <!-- fallback (boolean, null, object) -->
        <div v-else class="text-zinc-300">{{ String(value) }}</div>
      </div>
    </div>
```
